# PocketStudio Schema Documentation

## Overview

This schema defines the deployment form that customers see in Google Cloud Marketplace.

## Customer Inputs (Required)

| Field | Type | Description |
|-------|------|-------------|
| **Application Name** | string | Unique identifier for this deployment |
| **Namespace** | string | Kubernetes namespace |
| **Replicas** | integer | Number of instances (1-10) |
| **fal.ai API Key** | string | Customer's fal.ai account key |
| **GCS Bucket** | string | Cloud Storage bucket name |

## Auto-Detected

| Field | Source |
|-------|--------|
| **GCP Project** | Detected from cluster metadata |
| **Reporting Secret** | Generated by Marketplace |

## NOT Required

| Field | Reason |
|-------|--------|
| ~~Google API Key~~ | Using Workload Identity instead |
| ~~MongoDB URI~~ | Deployed in cluster |
| ~~Redis URI~~ | Deployed in cluster |

## Pricing Model

- **Metric:** Instance time (hours)
- **Rate:** $1.37 per hour per replica
- **Billing:** ~$1,000/month for 1 replica running 24/7

## Cluster Requirements

- **Minimum Nodes:** 2
- **CPU:** 2.5+ cores total
- **Memory:** 5GB+ RAM total
- **Workload Identity:** Required

## Customer Responsibilities

1. Create GCS bucket before deployment
2. Provide valid fal.ai API key
3. Pay for:
   - Software license (via Marketplace)
   - GKE cluster (to Google)
   - fal.ai usage (to fal.ai)
   - Storage & APIs (to Google)
